Математическое обеспечение информационной системы (фрагмент для документации)
================================================================================

Ниже перечислены соотношения, реализованные в коде приложения и пригодные для
включения в раздел «Математическое обеспечение ИС» (координаты панорамы,
масштабирование, прогресс тура, обработка изображений).

--------------------------------------------------------------------------------
1. Связь азимута и угла наклона с координатами на прямоугольной развёртке
--------------------------------------------------------------------------------

Используется линейное отображение углов в пиксели экрана (модель экваториальной
развёртки панорамы без учёта кривизны сферы — достаточно для интерфейса).

Обозначения:
  W, H — ширина и высота области отображения (или изображения) в пикселях;
  x, y — координаты точки на области (горизонталь, вертикаль);
  α — азимут, градусы, диапазон удобно приводить к [0, 360);
  φ — угол наклона (pitch), градусы, диапазон [-90, 90].

Прямое отображение (из долей координат в углы), как при клике по превью в админке:
  α = (x / W) · 360
  φ = (y / H) · 180 − 90

Обратное отображение (из углов в пиксели для позиционирования маркеров):
  x = ((α mod 360) / 360) · W
  y = ((clamp(φ, −90, 90) + 90) / 180) · H

Реализация: front/static/js/admin.js (размещение и редактирование маркеров),
front/static/js/tour_viewer.js (функции recalcScaledTile, azimuthToPx,
updateMarkersPosition).

--------------------------------------------------------------------------------
2. Масштабирование панорамы под высоту контейнера
--------------------------------------------------------------------------------

Пусть w_pano, h_pano — натуральная ширина и высота изображения панорамы,
h — высота контейнера просмотра. Масштабирование с сохранением пропорций по
вертикали:

  s = h / h_pano

Эффективная ширина «плитки» для горизонтальногой прокрутки (в пикселях):

  W_tile = w_pano · s

Реализация: front/static/js/tour_viewer.js, функция recalcScaledTile.

--------------------------------------------------------------------------------
3. Горизонтальная позиция по азимуту на масштабированной плитке
--------------------------------------------------------------------------------

  x_α = ((α mod 360) / 360) · W_tile

Реализация: front/static/js/tour_viewer.js, функция azimuthToPx.

--------------------------------------------------------------------------------
4. Циклический сдвиг панорамы (бесшовная горизонтальная прокрутка)
--------------------------------------------------------------------------------

Пусть s — накопленное смещение в пикселях, W_tile — ширина плитки. Приведение
смещения к основному периоду [0, W_tile) с корректной работой при отрицательных s:

  s' = ((s mod W_tile) + W_tile) mod W_tile

Реализация: front/static/js/tour_viewer.js, вычисление offsetMod для
background-position.

--------------------------------------------------------------------------------
5. Вертикальное положение маркера по углу наклона
--------------------------------------------------------------------------------

Для высоты контейнера H:

  y = ((clamp(φ, −90, 90) + 90) / 180) · H

Реализация: front/static/js/tour_viewer.js, updateMarkersPosition.

--------------------------------------------------------------------------------
6. Инерция прокрутки (дискретная модель по кадрам)
--------------------------------------------------------------------------------

При движении указателя оценивается скорость (в коде — приближение «пикселей
на кадр» через отношение смещения к интервалу времени и эмпирический множитель).

Затухание скорости с постоянным коэффициентом μ на каждом кадре (в реализации
μ = 0,92):

  v_{k+1} = μ · v_k
  s_{k+1} = s_k + v_k

Остановка, когда |v| < порога.

Реализация: front/static/js/tour_viewer.js, функции onPointerMove, startInertia.

--------------------------------------------------------------------------------
7. Вписывание мини-карты в прямоугольник с сохранением пропорций
--------------------------------------------------------------------------------

Натуральные размеры изображения w, h; максимально допустимые размеры области
W_max, H_max. Единый масштаб (uniform scaling):

  λ = min(W_max / w, H_max / h)

Итоговые размеры отображения:

  w' = w · λ,  h' = h · λ

Реализация: front/static/js/tour_viewer.js, функция fitMinimapBox.

--------------------------------------------------------------------------------
8. Прогресс прохождения тура (доля просмотренных информационных меток)
--------------------------------------------------------------------------------

N_total — число информационных меток в туре, N_viewed — число уникальных
просмотренных пользователем меток (по данным учёта). Процент в целых числах:

  P = round(100 · N_viewed / N_total),  если N_total > 0,
  P = 0,  если N_total ≤ 0.

Реализация: map_api/serializers.py (TourSerializer.get_progress_percent),
map_api/views.py (действие mark_viewed).

--------------------------------------------------------------------------------
9. Обрезка изображения после загрузки (сервер)
--------------------------------------------------------------------------------

В map_api/utils.py функция apply_crop выполняет поворот изображения (учёт знака
угла: Cropper.js задаёт поворот по часовой стрелке, Pillow — против) и
затем вырезает прямоугольник с границами, вычисленными из координат обрезки
(x, y, width, height) с округлением до целых пикселей и ограничением границ
размерами изображения. Для раздела МО при необходимости можно дополнительно
описать аффинное преобразование поворота и переход к целочисленным границам кропа.

Реализация: map_api/utils.py, apply_crop.
